2026.04 — UX 구조 분석 및 설계 방향 제안

1. AI 공간정보 플랫폼 UI 트렌드

2025~2026년 주요 GIS/AI 플랫폼이 공통적으로 채택하고 있는 UI 패턴입니다.

지도 퍼스트

대시보드가 아닌 지도가 기본 진입점. 필요한 정보만 플로팅 HUD로 표출.

레이어 상시 노출

레이어 패널은 항상 접근 가능. 분석·검색 등은 독립 패널로 분리하여 동시 작업 가능.

프로그레시브 디스클로저

모든 기능을 한 번에 노출하지 않음. 사용자 행동에 따라 점진적으로 상세 정보 표출.

역할 기반 UI

분석가·관리자·열람자별로 메뉴와 기능을 다르게 구성. 불필요한 기능은 숨기는 것이 표준.

2. 유사 서비스 벤치마킹

Land-XI는 AI 모델 관리(학습·배포) + 분석 실행 + 결과 지도 조회가 결합된 플랫폼입니다. 이에 맞는 비교 대상을 선정했습니다.

FlyPix AI — 위성/드론 영상 AI 분석 플랫폼

FlyPix AI — 지도 풀스크린 + 좌측에 AI 분석 결과(객체 탐지 목록) 상시 노출

  • 구조: 지도가 기본 화면, 좌측 패널에 AI 탐지 결과(객체 수, 면적)가 상시 표시
  • 네비게이션: 상단 바 최소화 (프로젝트명 + 모드 전환만), 기능은 좌측 패널과 우측 도구로 분리
  • 분석 워크플로우: 영상 업로드 → 모델 선택 → 분석 실행이 단일 워크스페이스 내에서 완결

출처: FlyPix AI Platform

Roboflow — 컴퓨터 비전 모델 관리·배포 플랫폼

Roboflow — 모델 모니터링 대시보드 (요청 수, 평균 신뢰도, 추론 시간 KPI)

Roboflow — 프로젝트/워크플로우/모니터링 탭 구조

  • 대시보드: KPI 카드(요청 수·신뢰도·추론시간) + 모델별 추론 이력 테이블
  • 네비게이션: 상단에 Projects / Workflows / Monitoring / Devices 4개 탭만 배치
  • 분석 파이프라인: 데이터 업로드 → 라벨링 → 학습 → 배포 → 모니터링 전체를 하나의 플랫폼에서 관리

출처: Roboflow Blog — Model Monitoring

Felt — 지도 기반 협업 플랫폼 (UI 구조 참고)

Felt — 인터페이스 구조도: Toolbar(상단) + Legend(좌측 상시) + Map(중앙) + Detail Panel(우측, 필요 시)

  • 핵심: 레이어(Legend)가 좌측에 상시 노출, 상세 패널은 클릭 시에만 우측에 표시
  • 지도와 패널이 공존: 탭 전환 없이 여러 패널을 동시에 볼 수 있는 구조

3. 탑 네비게이션 구조

관리자 뷰 — 수평 네비게이션 9개 메뉴

문제: 메뉴 수와 가로 해상도

현재 관리자 화면에 9개 메뉴가 수평으로 나열되어 있습니다. 탑 네비게이션에서 메뉴가 정상적으로 표시되려면 각 메뉴 아이템에 아이콘(16px) + 텍스트(한글 4자 기준 약 56px) + 좌우 패딩(24px) = 약 96px이 필요합니다.

메뉴 수필요 너비대응 해상도
5개~480px태블릿 이상 대응 가능
7개~672px1024px 이상 권장
9개~864px1280px 이상 필수, 로고+우측 영역 포함 시 1440px+
우려 사항 9개 메뉴를 수평 나열하면 1280px 미만 해상도에서 메뉴가 잘리거나 줄바꿈이 발생합니다. Nielsen Norman Group의 연구에 따르면 탑 네비게이션의 권장 메뉴 수는 5~7개이며, 이를 초과할 경우 사용자의 탐색 효율이 감소합니다. 실제로 지자체 뷰(6개)와 비교했을 때 관리자 뷰(9개)는 시각적 밀도가 높아 어떤 메뉴가 핵심인지 구분이 어렵습니다.

지자체 뷰 — 6개 메뉴, 상대적으로 여유 있는 구성

대안 핵심 메뉴(대시보드·지도서비스·보고서)와 관리 메뉴(영상·학습·모델 등)를 2단 그룹핑하거나, 관리 메뉴를 드롭다운/사이드바로 분리하면 5~6개 수준으로 축소 가능합니다. 이렇게 하면 1024px 해상도에서도 안정적으로 대응됩니다.

4. 메뉴 하위 사이드바 탭 구조

지도서비스 — 좌측 사이드바에 레이어/분석/라벨링/검색 4개 탭

구조: 네비게이션 → 사이드바 → 탭

현재 사용자가 특정 기능에 도달하기까지의 계층 구조:

탑 메뉴(지도서비스) → 좌측 사이드바 → 탭(레이어/분석/라벨링/검색) → 기능 내용

총 3단계 계층입니다.

우려 사항 1 — 컨텍스트 유실 탭 전환 시 이전 탭의 상태가 사라집니다. 예를 들어 레이어를 조정하다가 분석 탭으로 이동하면, 레이어 설정 화면이 완전히 사라져 두 기능을 동시에 참조할 수 없습니다. GIS 작업에서는 레이어를 보면서 분석을 실행하거나, 검색 결과를 보면서 레이어를 조정하는 것이 일반적인 워크플로우입니다.

분석 탭 전환 시 레이어 설정이 사라짐

우려 사항 2 — 탭의 성격 혼재 레이어 관리(상시 필요)와 라벨링(전문 기능)이 동일 탭 레벨에 있습니다. 사용 빈도와 성격이 전혀 다른 기능이 같은 우선순위로 배치되면 사용자가 "이 탭이 나한테 필요한 건가?" 판단하는 데 불필요한 인지 부하가 발생합니다.
장점 사이드바 영역을 재활용하므로 화면 공간을 절약할 수 있고, 각 탭의 내용이 독립적이라 개별 기능의 UI가 깔끔하게 유지됩니다. 분석 탭의 "카드 선택 → 범위 설정 → 실행" 워크플로우는 직관적으로 잘 설계되어 있습니다.
대안 레이어 패널을 상시 노출하고, 분석·검색은 별도 플로팅 패널로 분리하는 방식이 GIS 플랫폼의 표준입니다 (ArcGIS Online, QGIS, Felt). 이렇게 하면 레이어를 보면서 동시에 분석을 실행할 수 있고, 라벨링 같은 전문 기능은 별도 모드로 진입하는 구조가 됩니다.

5. 대시보드 vs 지도 진입점

현재 로그인 후 기본 화면은 대시보드입니다.

현황 대시보드에 KPI 카드, 최근 분석, 스토리지 현황, 공지사항, 프로젝트 목록이 한 화면에 배치되어 있습니다. 이는 프로젝트 관리 중심 사용자에게 적합하지만, 지도 기반 업무를 수행하는 사용자에게는 지도까지 한 번 더 클릭해야 하는 불필요한 동선이 됩니다.
기본 진입점적합한 사용자사례
대시보드프로젝트 관리자, 데이터 사이언티스트Weights & Biases, MLflow
지도 풀스크린공간정보 분석가, 현장 담당자, 열람자ArcGIS Online, Google Earth Engine, Felt
제안 사용자 역할에 따라 기본 진입점을 분기할 것을 권장합니다. 관리자는 대시보드, 열람자/분석가는 지도서비스로 직접 진입. Land-XI의 권한별 뷰 분기 구조를 활용하면 구현 가능합니다.

6. 권한별 UI 분기

Land-XI는 이미 관리자/LX/지자체 3단계 권한별 화면 분기를 구현하고 있습니다. 이 구조는 매우 긍정적입니다.

LX 뷰 — 관리자 대비 보고서·통계 메뉴 제거

권한메뉴 수빠른 실행
관리자9개4개 (지도·영상·프로젝트·라벨링)
LX7개4개
지자체6개2개 (지도·영상)
좋은 구조 지자체 뷰에서 학습데이터·모델학습 등 전문 메뉴가 제거되고 빠른 실행이 2개로 축소된 것은 역할 기반 UI의 좋은 사례입니다. 이 패턴을 우리 서비스에 그대로 활용할 수 있습니다.

7. 디자인 시스템

장점 카드 컴포넌트, 뱃지, 테이블, 버튼(Primary/Secondary) 스타일이 전반적으로 일관되며, 다크 네이비 헤더 + 라이트 배경 + 화이트 카드의 3단 색상 체계가 안정적입니다.
우려 사항 아이콘이 대부분 placeholder(빈 사각형)로 표시되고 있어 실제 사용 시 시각적 구분력이 떨어질 수 있습니다. 최종 구현 시 의미 있는 아이콘으로 교체가 필요합니다.
